BYTE

It is an integer type, it stores only signed values they can be positive (or) negative. Byte accepts the value in ranges between -128 to 127. Size is 1 byte. We declare byte because the time required for execution is less compared to other data types.

Example :

byte 50 takes less time for manipulation compared to int 50 because, the memory size of byte is less compared to int.

Memory size of byte ranges from -128 to 127.

Memory size of int ranges from -2,147,483,648 to 2,147,483,647.

When we declare byte, compiler arranges -128 to 127 spaces in memory for our identifier.

TYPE SIZE MINIMUM VALUE MAXIMUM VALUE
byte one byte -128 127

Program:

public class Data{
public static void main(String[] args)
{
byte max=127;
byte min=-128;
byte num=50;
System.out.println("MAXIMUM VALUE OF BYTE DATA TYPE IS:" +max);
System.out.println("MINIMUM VALUE OF BYTE DATA TYPE IS:" +min);
System.out.println("VALUE OF BYTE DATA TYPE IS:" +num);
}
}

Output:

In the above program we can see the maximum and minimum ranges of the byte data type so that we can declare an value in between -128 to 127 such as 50, 100, -27 e.c.t. we should not exceed 127 (or) -128.

If we exceed the range of byte data type it gives compiler error, see below program:

public class Data{
public static void main(String[] args)
{
byte num=150;
System.out.println("VALUE OF BYTE DATA TYPE IS:" +num);
}
}

Output:

Microsoft Windows [Version 6.1.7600]
Copyright (c) 2009 Microsoft Corporation. All rights reserved.
C:\Users\fortutorials>d:
D:\>cd fortutorials
D:\fortutorials>javac Data.java
Data.java:4: possible loss of precision found : int
required: byte
byte num=150;
^
1 error
D:\fortutorials>

We had given the value 150 then it gives compiler error. The given value is int data type but required byte.
