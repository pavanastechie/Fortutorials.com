JAVA IDENTIFIERS

Note: we explained the identifier for naming the class and variables, there are many term in Java where we use identifiers.

Identifiers:

Identifiers are the smallest individual units in a Java program, the person who is writing program can declare identifiers, we can declare any name but follows below given rules. Identifiers are used for naming classes, methods, variables, objects, labels, packages and interfaces in a program.

RULE-3:

It should not declare reserved word (or) keyword as an identifier, there are some reserved words in Java, they are declared by oracle Java organization, we shouldn't use these words as identifiers.

Examples of reserved (or) keywords are static, public e.c.t.

USING UPPER, LOWER, CAMEL CASES in IDENTIFIERS:

We can declare identifiers in any case (upper,lower,camel) compiler doesn't raise error, But we have to follow a notation for identification identifiers from one to be other.(class name and variable name).

Classes and Interfaces - starting letter to the class, interfaces are started with an uppercase letter for multiple worded identifiers use camel case.

Camel case: the uppercase letters in the middle of identifier name

Example:

Print, Data, LinearAdd (camelcase) e.c.t

VARIABLES, METHOD -lowercase is used in the declaration of variables and methods and we can use camel case for multiple worded identifiers.

Example:

rollno, range, number, roleList (camelcase).

CONSTANTS: We can name the constants with uppercase, underscore (_) to separate the multiple words used in constants.

Example:

ADD_MY_NUMBER, ADD_MY_LIST e.c.t
